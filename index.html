JavaScriptの基本解説

Javascriptの特徴

JavaScriptには、それぞれ以下の特徴がある。

● 大文字と小文字を区別する

● セミコロンで区切る

● 予約語が存在する

● strict mode （実行モード）が存在する

それぞれ順番に解説する。

1.大文字と小文字を区別する

JavaScriptは、大文字と小文字を区別する。そのため、変数名、関数名、オブジェクトプロパティ名などの識別子は、大文字と小文字を区別する必要がある。

例えば、次のような変数名はすべて異なる関数名として扱われる。


                var ThisVariable = "国語"
                var THISVARIABLE = "数学"
                var thisvariable = "理科"

同様に、次のような関数名も全て異なる関数として扱われます。


                function ThisFunction() {
                  return "国語"
                }
                function THISFUNCTION() {
                  return "数学"
                }
                function thisfunction() {
                  return "理科"
                }

ただし、JavaScriptのキーワード（予約語）は全て小文字で書く必要がある。

キーワードはJavaScriptの構文で特別な意味を持つため、大文字で書くとエラーが発生する可能性がある。

一般的にJavaScriptのコードを書くときは、識別子を一貫して大文字か小文字かで統一することが重要。

これによって、コードの可読性が向上し、誤解を招くようなエラーを避けられる。

2.セミコロンで区切る

JavaScriptでセミコロン（;）は、文の終わりを示すために使用される。つまり、1つの文が終わったら、必ずセミコロンをつける必要がある。

ただし、JavaScriptではセミコロンを省略することもできる。具体的には、次のような場合である。

●改行が文の終わりを示す場合

●文がブロック（{ }）で囲まれている場合

ただし、セミコロンを省略するとコードが予期しない動作をする可能性があるため、一般的にはセミコロンをつけることが推奨されている。

また、複数の文を1行に書く場合には、それぞれの文の終わりにセミコロンをつける必要がある。

3. 予約語が存在する

JavaScriptには「予約語」と呼ばれる、言語自体で特別な意味を持つ単語が存在する。これらの単語は、変数名や関数名などの識別子として使用することができないようになっている。

JavaScriptの予約語には、以下のようなものがあります。

● break: ループまたはswitch文から抜け出すために使用される。

● continue: ループ内の次のイテレーションに進むために使用される。

● extends: クラスを継承するために使用される。

● function: 関数を定義するために使用される。

● import: モジュールをインポートするために使用される。

● new: オブジェクトを生成するために使用される。

4.strict mode（実行モード）が存在する

JavaScriptでは、「strict mode（実行モード）」という、JavaScriptコードをより厳密に解釈するための機能が存在する。strict modeは、コードの先頭で「”use strict”;」と記述することで有効になる。

strict modeを使うことで、以下のような制限が行われる。

● グローバル変数の暗黙的な宣言の禁止

● 厳密な関数有効範囲のルール

● 重複する引数名やプロパティ名の禁止

● eval関数の厳密な動作

● セキュリティの強化

Strict modeは、コードの保守性を高め、バグを減らし、安全性を向上させるための便利な機能である。しかし、過去のコードとの互換性が損なわれる場合があるため、注意して使用する必要がある。
